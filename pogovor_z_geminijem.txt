Preklapljanje naslova in piškotki

User prompt: ali preklapljanje na kanadski naslov spletne strani odstrani ''cookiese''?
Response: Kratki odgovor: Ne.
Preklapljanje na kanadski naslov (ali katero koli drugo regijo) ne izbriše obstoječih piškotkov iz vašega brskalnika. Lahko pa vpliva na to, kateri piškotki se v tistem trenutku uporabljajo.
Kaj se dejansko zgodi?
Vse je odvisno od tega, kako natančno se spremeni spletni naslov (URL):
Sprememba celotne domene (npr. iz stran.com na stran.ca): Brskalniki zaradi varnostnih pravil (t.i. Same-Origin Policy) ne dovolijo, da bi ena domena brala piškotke druge. Ko obiščete .ca različico, ta ne more videti piškotkov od .com različice. Vendar pazi: piškotki od .com še vedno ostanejo shranjeni v brskalniku. Ko se boste vrnili na .com, vas bodo tam še vedno "prepoznali". Kanadska stran bo medtem preprosto ustvarila svoj, nov komplet piškotkov.
Sprememba le lokalnega dela naslova (npr. iz stran.com na [stran.com/ca](https://stran.com/ca)): Ker je glavna domena (stran.com) ostala popolnoma ista, brskalnik obdrži in uporablja vse obstoječe piškotke. Preklop na kanadsko podstran v tem primeru ne spremeni popolnoma ničesar.
Kako dejansko očistiti piškotke?
Če se želite piškotkov znebiti, ker želite ponastaviti stran, skriti svojo lokacijo ali pa vam spletno mesto javlja napako, preklapljanje med regijami ne bo pomagalo. Uporabite raje naslednje metode:
Uporabite način brez beleženja zgodovine (Incognito): Najhitrejša pot, če želite začeti z "svežim" profilom brez predhodne zgodovine.
Klik na ikono ključavnice: V vrstici za vnos spletnega naslova (URL) kliknite na ikono ključavnice (ali ikono nastavitev) na levi strani. Izberite Piškotki in podatki o spletnem mestu (Cookies and site data) ter jih izbrišite samo za to specifično stran.
Bližnjica za brisanje: Pritisnite kombinacijo tipk Ctrl + Shift + Delete (Windows) ali Cmd + Shift + Delete (Mac), da odprete meni za brisanje podatkov brskanja.

User prompt: kako napišem try/except block, če želim preveriti, ali python, ki ga pošiljam na spletno stran dejansko dobi html kodo?
Response: Za varno pridobivanje HTML kode s spletne strani v Pythonu je najbolje uporabiti knjižnico requests. Da bo koda res robustna, moramo preveriti dve stvari:
Ali je bila omrežna zahteva uspešna (da stran sploh obstaja in ni javila napake 404 ali 500).
Ali je vrnjena vsebina dejansko HTML (in ne npr. slika, PDF ali JSON datoteka).
Spodaj je primer šolskega try/except bloka, ki pokrije vse te scenarije.
Koda: Robusten try/except blok
import requests

url = "https://example.com"

try:
    # 1. Pošljemo zahtevo (nastavimo timeout, da program ne obvisi, če stran neodzivna)
    response = requests.get(url, timeout=10)
    
    # 2. Preverimo HTTP status (sproži napako, če je status 4xx ali 5xx)
    response.raise_for_status()
    
    # 3. Preverimo, ali je glava (Header) odziva res "text/html"
    content_type = response.headers.get('Content-Type', '')
    if 'text/html' not in content_type:
        # Če ni HTML, sami sprožimo ValueError, ki ga ujame spodnji except
        raise ValueError(f"Nismo dobili HTML kode, ampak: {content_type}")
        
    # Če pridemo do sem, je vse v redu
    html_koda = response.text
    print("HTML koda je uspešno prejeta!")
    print(html_koda[:200]) # Izpis prvih 200 znakov za test

except requests.exceptions.HTTPError as errh:
    print(f"HTTP Napaka (npr. 404 ali 500): {errh}")

except requests.exceptions.ConnectionError as errc:
    print(f"Napaka pri povezavi (napačen URL ali ni interneta): {errc}")

except requests.exceptions.Timeout as errt:
    print(f"Časovna omejitev je potekla (strežnik se ne odziva): {errt}")

except requests.exceptions.RequestException as err:
    print(f"Splošna napaka pri zahtevi: {err}")

except ValueError as err_vsebina:
    print(f"Napaka v tipu podatkov: {err_vsebina}")


Kako to deluje?
response.raise_for_status(): To je ključna funkcija. Če spletna stran vrne kodo 200 OK, se ne zgodi nič. Če pa vrne npr. 404 Not Found ali 500 Internal Server Error, bo ta vrstica namerno sprožila izjemo (HTTPError), ki jo nato ujame ustrezen except.
Preverjanje Content-Type: Spletni strežniki v glavi odgovora vedno povedo, kaj pošiljajo. Z ukazom 'text/html' not in content_type poskrbimo, da nas program pravočasno opozori, če smo namesto spletne strani ponesreči prenesli sliko ali JSON datoteko.
timeout=10: Vedno določite časovno omejitev (v sekundah). Če se strežnik "zacikla", bo vaš program po 10 sekundah vrgel napako Timeout in se ne bo zagozdil za vedno.

User prompt: katero knjižnico naj uporabim, če želim delati FFT na podatkih delnic. Ali je scipy.fft ustrezna in kako jo naložim?
Response: Da, scipy.fft je odlična in povsem ustrezna izbira!
Zlati standard za izvajanje hitre Fourierove transformacije (FFT) v Pythonu sta dve knjižnici: SciPy (scipy.fft) in NumPy (numpy.fft).
Za analizo delnic je SciPy celo malenkost boljša izbira, saj je njen algoritem v določenih situacijah hitrejši in ponuja več naprednih funkcij za obdelavo signalov (npr. enostavno odstranjevanje trendov, kar je pri delnicah ključno).

Hitri primer: Kako jo uporabiti na delnicah
Podatki o delnicah imajo linearen trend (rastejo ali padajo skozi čas), kar lahko popači rezultate FFT. Zato je pred izvedbo FFT priporočljivo podatke detrendati (odstraniti trend) ali pa uporabiti dnevne donose.
Tukaj je preprost primer, kako preveriti cikle v ceni delnice:
import numpy as np
import pandas as pd
import yfinance as yf
from scipy.fft import fft, fftfreq
from scipy.signal import detrend

# 1. Prenos podatkov (npr. Apple za zadnji dve leti)
data = yf.download("AAPL", start="2024-01-01", end="2026-01-01")
cene = data['Close'].values.flatten()

# 2. Odstranitev trenda (nujno za delnice!)
cene_brez_trenda = detrend(cene)

# 3. Izračun FFT
N = len(cene_brez_trenda)
fft_vrednosti = fft(cene_brez_trenda)
frekvence = fftfreq(N, d=1) # d=1 pomeni 1 dan

# Ker je FFT simetričen, vzamemo le pozitivno polovico spektra
pozitivni_del = N // 2
amplitude = np.abs(fft_vrednosti[:pozitivni_del])
dejanske_frekvence = frekvence[:pozitivni_del]

print("FFT je uspešno izračunan!")


Na kaj morate biti pozorni?
Borzni podatki imajo eno posebnost: vikendi in prazniki manjkajo. To pomeni, da vaša časovna vrsta nima popolnoma enakomernih časovnih razmikov (med četrtkom in petkom je 1 dan, med petkom in ponedeljkom pa 3 dnevi). Za osnovno analizo večina matematikov to preprosto ignorira in predpostavi, da je "1 korak = 1 trgovalni dan", kar v praksi deluje povsem solidno.

Ker praviš, da delaš na ustvarjanju "povprečne delnice" in da je koda zadnjič delovala, danes pa ne, se je najverjetneje spremenilo nekaj v samih vhodnih podatkih. Poglejva tri najpogostejše krivce, zakaj se je to zgodilo prav danes:
Krivec 1: Uporaba .fillna(0) na napačnem mestu
Ko združuješ več delnic skupaj, da bi izračunal njihovo povprečje, se pogosto zgodi, da kakšna delnica nima podatka za določen dan (npr. zaradi praznikov ali različnih borz). Pri združevanju tabel (npr. z pd.merge ali pd.concat) nastanejo prazna polja (NaN).
Če si nekje v kodi napisal nekaj takega:
# POZOR: To spremeni VSE prazne vrednosti v tabeli v 0, tudi datume!
df = df.fillna(0) 


Če je funkcija .fillna(0) ulovila stolpec z datumi, so vsi prazni datumi postali 0. Ko je koda nato izvedla pd.to_datetime(), so te ničle postale 1970-01-01.
Krivec 2: Nova prazna vrstica v CSV datoteki (past s transponiranjem .T)
Tvoja koda uporablja trik s transponiranjem: CENE_SP_500.T.reset_index(). Če je spletna stran, s katere prenašaš podatke, danes v CSV datoteko dodala eno prazno vrstico na koncu (kar se pogosto zgodi ob vikendih ali ob prehodu v nov teden), je ta prazna vrstica po transponiranju postala nov stolpec/vrstica brez imena. Pandas jo je prebral kot NaN ali 0 in jo pretvoril v leto 1970.
Krivec 3: Napačna pretvorba indeksa
Če si pri združevanju delnic ponesreči resetiral indeks in nato poskušal pretvoriti številke indeksa (0, 1, 2, 3...) v datume, bo prva vrstica (indeks 0) postala leto 1970, naslednja (indeks 1) pa sekundo ali dan kasneje.

Da bi to avtomatiziral, bova tvoje podatke združila v en Pandas DataFrame, izračunala to razmerje (Donos / Tveganje), jih razvrstila in nato z zanko dodala oznake za top n podjetij.

Dopolnjena koda z avtomatskim označevanjem
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# --- 1. Nastavi število podjetij, ki jih želiš označiti ---
n = 5  # To številko lahko poljubno spremeniš (npr. 3, 10, 15...)

# --- 2. Izračun donosov in tveganja (tvoja koda) ---
donosi = CENE_500_DELNIC.pct_change().mean() * 252
tveganje = CENE_500_DELNIC.pct_change().std() * (252**0.5)

# --- 3. Združevanje in iskanje najboljših ---
# Podatke damo v skupno tabelo, kjer so indeksi imena podjetij (npr. AAPL, MSFT...)
df_analiza = pd.DataFrame({
    'Donos': donosi,
    'Tveganje': tveganje
})

# Izračunamo razmerje: višje kot je, bolj smo v levem zgornjem kotu
df_analiza['Razmerje'] = df_analiza['Donos'] / df_analiza['Tveganje']

# Izberemo top N podjetij z najvišjim razmerjem
najboljša_podjetja = df_analiza.sort_values(by='Razmerje', ascending=False).head(n)

# --- 4. IZRIS GRAFA ---
plt.figure(figsize=(11, 6))

# Narišemo vse delnice v sivi barvi, da graf ni prenatrpan
plt.scatter(df_analiza['Tveganje'], df_analiza['Donos'], alpha=0.4, color='gray', label='Ostale delnice')

# Posebej narišemo top N delnic v rdeči barvi, da izstopajo
plt.scatter(najboljša_podjetja['Tveganje'], najboljša_podjetja['Donos'], color='red', s=60, label=f'Top {n} delnic')

# Z zanko se sprehodimo skozi najboljša podjetja in jim dodamo besedilo (ime)
for ime_podjetja, vrstica in najboljša_podjetja.iterrows():
    plt.annotate(
        ime_podjetja, 
        (vrstica['Tveganje'], vrstica['Donos']),
        textcoords="offset points", 
        xytext=(5, 5),      # Zamik besedila (5 pik desno, 5 pik gor od pike)
        ha='left',          # Poravnava besedila
        color='black', 
        fontweight='bold',
        fontsize=9
    )

# Oznake in naslovi
plt.xlabel('Tveganje (Volatilnost)')
plt.ylabel('Povprečen letni donos')
plt.title('Risk vs. Reward (Izpostavljenih top N podjetij)')
plt.legend()
plt.grid(True, alpha=0.3)

plt.show()
